# CLAUDE.md — 프로젝트 공통 규칙

> 모든 에이전트는 작업 전 이 파일을 반드시 참조한다.

---

## 문서 구조

```
Oh-My-Pomodoro/
├── README.md              # 프로젝트 소개 (외부 공개용)
├── CLAUDE.md              # 이 파일. 모든 에이전트 공통 규칙.
├── TEAM_LEAD.md           # 팀장 운영 문서 (팀장만 수정)
└── docs/                  # 프로젝트 문서 디렉토리
    ├── SPEC.md            # 기획서 (제품 요구사항 정의)
    └── <module>/          # 모듈별 서브폴더 (기능 구현 시 생성)
        └── DESIGN.md      # 해당 모듈의 설계 문서
```

### 문서 배치 규칙

| 문서 종류 | 위치 | 설명 |
|---|---|---|
| 프로젝트 소개 | `README.md` (루트) | 외부 공개용. 프로젝트 개요만 기술. |
| 에이전트 공통 규칙 | `CLAUDE.md` (루트) | 모든 에이전트가 참조하는 공통 규칙. |
| 팀장 운영 문서 | `TEAM_LEAD.md` (루트) | 워크플로우, 리뷰 기준. 팀장만 수정. |
| 기획서 | `docs/SPEC.md` | 제품 요구사항, MVP 정의. |
| 모듈 설계 문서 | `docs/<module>/DESIGN.md` | 해당 모듈의 아키텍처, 인터페이스, 제약사항. |

### 문서 참조 원칙

- 에이전트는 **할당된 프롬프트에 명시된 문서만** 읽는다.
- 프로젝트 전체 문서를 탐색하지 않는다.
- 팀장이 에이전트 할당 시 필요한 문서 경로를 프롬프트에 포함한다.

## 코드 작성 규칙

- 변수/함수명은 의미를 명확히 드러내는 이름을 사용한다.
- 주석은 "왜(Why)"가 명확하지 않은 경우에만 작성한다.
- 하드코딩된 값은 상수로 분리한다.
- 불필요한 추상화를 만들지 않는다.
- 사용하지 않는 코드는 주석 처리가 아닌 삭제한다.
- OWASP Top 10 보안 취약점을 방지한다.
